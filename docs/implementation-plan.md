## 恢复实施前要先准备的两件事

1. **填 `.env`**：`cp .env.example .env`，填 `ANTHROPIC_API_KEY`（Claude）+ 一个 embedding key（务实型隐私推荐 `OPENAI_API_KEY`）。`ANTHROPIC_MODEL` 默认 `claude-sonnet-4-6`，可按 Anthropic 当前推荐模型调整。`HOMEBRAIN_DIR` 会作为 `GBRAIN_HOME` 传给 gbrain。已 gitignore。
2. **装 gbrain**（Slice 0）：
   ```bash
   git clone https://github.com/garrytan/gbrain.git ~/gbrain
   cd ~/gbrain && git checkout v0.42.52.0   # 若 tag 缺失，按 Slice 0 实测记录锁 commit
   bun install && bun link                   # 提供全局 gbrain 可执行
   gbrain init --pglite                       # 本地 PGLite brain，无需 Docker
   gbrain doctor                              # 健康检查
   ```
   **别用 `npm i -g gbrain`**（npm 上是抢注包）；**别用 `bun install -g github:...`**（postinstall 会被拦截装坏）。gbrain 当外部可执行依赖，不进 `dependencies`。

## 开工前必须先验证的 3 个未知数

按优先级（每个都可能改变后续实现）：

1. **gbrain 能力确认**：`gbrain query` 默认用哪个 provider/model、能否把 synthesis 指到 Claude、云 embedding（OpenAI 等）怎么配。
2. **PGLite 上 dream-cycle / consolidate 到底能跑哪些 phase**——跑不动就退化为"检索 + Claude 现场总结"，或直接上 Postgres。同时实测 serve/sync/write 写锁互斥行为。
3. **飞书最小收发闭环**：`lark-cli event consume` 收一条 → `im send` 回一条；确认 bot 的 IM 读写权限、事件订阅 scope、bot open_id、图片下载权限、子进程崩溃自愈。

## Slice 0 必须产出的 gbrain contract

Slice 0 不是只手工跑通命令，而是把 gbrain 的外部可执行 interface 固化成可复跑的 contract。当前实测见 [Slice 0 gbrain contract 实测记录](./slice0-gbrain-contract.md)。产出：

1. **命令矩阵**：记录 `capture`、`query`、`search`、`sync`、`doctor`、dream-cycle 相关命令在 `v0.42.52.0` + PGLite 下的真实参数、退出码、stdout/stderr、是否支持 `--json`。已确认：`capture --json` 输出 JSON；`search/query` 帮助里没有 `--json`，输出文本。
2. **fixture**：保存关键输出样例，作为 `packages/homebrain/src/gbrainCli.ts` 的 contract 测试输入。fixture 里只放脱敏样例，不放真实家庭聊天。
3. **contract 测试**：围绕 `gbrainCli` module 验证命令构造、JSON 字段归一、文本 search/query 解析、错误信息、不可用命令的 fallback 决策。
4. **fallback 决策**：如果时间检索、USER.md 写入、dream-cycle 某 phase 在 PGLite 不可用，必须在这里记录替代路径（例如 `search + 本地 occurredAt 过滤`、`检索 + Claude 现场总结`、或切 Postgres）。当前已知：无 embedding key 时 `sync` 会因 provider 缺失失败，开发模式可用 `runDreamCycle({ noEmbed: true })` 跳过 sync embedding；sync 仍要求 source 有 `local_path`，`GBRAIN_SOURCE_PATH` 存在时 homebrain 会自动 `sources list/add` 注册缺失 source，但不会对已存在的 `default` source 做 destructive remove/re-add。`gbrain dream --json` 即使 exit 0 也可能返回 `status: "partial"`，wrapper 已按 report status 判断成败。
