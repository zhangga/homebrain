# CLAUDE.md

面向家庭的长期记忆 AI 助手。本地优先、单家庭、自托管。详见 `docs/`：
[架构](docs/architecture.md) · [实施计划](docs/implementation-plan.md)。

## 技术栈与命令

- **TS + Bun**（对齐 gbrain）。Bun ≥ 1.3.10。
- Monorepo（Bun workspaces）：`packages/homebrain` + `packages/homeagent`。
- 常用：
  - `bun test` —— 全部测试
  - `bun run typecheck`（= `tsc --noEmit`）
  - `bun run packages/homeagent/src/index.ts` 或 `make dev` —— 本地跑（CLI connector，不依赖飞书）
  - `make help` —— 看所有 make 目标

## 架构铁律（改动前必读）

- **单向依赖**：`homeagent → homebrain → gbrain(CLI)`。homeagent **永不**直接 spawn gbrain。
- **homebrain 是全项目唯一接触 gbrain 的地方**。集成边界 = gbrain **CLI 子进程**（库层没有 capture/query/dream 等"友好动词"，只在 CLI/MCP 层）。新增 gbrain 能力 = 在 `packages/homebrain/src/index.ts` 加方法，不要在 homeagent 里调 gbrain。
- **写操作必须串行化**（`packages/homebrain/src/serialize.ts`）：gbrain 底层 PGLite 是单写者，并发写会争锁报错。所有写经同一 FIFO 队列；读不受限。
- **connector 可插拔**：实现 `packages/homeagent/src/connectors/types.ts` 的 `Connector` 接口即可加平台（飞书/微信/QQ/CLI）。平台细节只能待在 adapter 里。

## 锁定决策（勿重新质疑）

| 维度 | 决策 |
|---|---|
| 基座 | `gbrain` 当**锁版本外部依赖**（`v0.42.52.0`），不 fork、不改内部 |
| 入口 | 飞书 Bot（首个 adapter；connector 可插拔） |
| 编排层 LLM | Claude（最新模型）|
| 隐私 | 务实型：聊天原文+记忆库只在本机；抽取/问答/总结/embedding 可调云 API（不长期留存）|
| 成员映射 | gbrain Model B：单 source + `partners/<slug>/` + `USER.md` |

## 陷阱

- **别 `npm i -g gbrain`**（npm 上是抢注包）；**别 `bun install -g github:...`**（postinstall 被拦截装坏）。装法见实施计划：clone + checkout tag + `bun link`。
- **gbrain 不进任何 `dependencies`**，它是外部可执行（类似依赖 `git`/`ffmpeg`）。
- gbrain 频繁 breaking change：升级前先跑 homebrain 的 CLI contract 测试（R1）。
- gbrain CLI 的 `--json` 输出形状与 PGLite 上 dream-cycle 行为已按 Slice 0 实测固化到 contract / 文档；飞书收发权限仍待真实凭据实测。新增外部能力前先补 contract，不要凭假设写死。
- 隐私：往云端发东西前对照"什么可以离开盒子"（见架构文档），原始聊天记录和记忆库**永不外发**。
